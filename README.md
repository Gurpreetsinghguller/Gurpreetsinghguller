## Software Architecture Fundamentals

I apply core architectural principles while designing systems:

### Four Pillars of Architecture

* **Abstraction** – Focus on essential features while hiding unnecessary details.
* **Encapsulation** – Restrict direct access to internal implementation details.
* **Decomposition** – Break complex systems into manageable modules and services.
* **Generalization** – Identify common patterns and create reusable components.

### Object Classification

* **Entity Objects** – Represent core business data and domain models.
* **Control Objects** – Handle business logic and application workflows.
* **Boundary Objects** – Manage interaction between external systems and the application (APIs, UI, gateways).
